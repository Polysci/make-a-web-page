LESSON ONE - Getting Started with HTML

THE INTERNET

      HTML is the basis of web documents and glues together other things like images and video. Internet Browsers make requests using a protocol called HTTP, and these requests are received by the Server where the files are stored. The Server sends these files back to your computer where your Browser interprets them into the web page you see.

HTML

      HTML makes up the greater portion of what's on the internet. It includes Text Content, as well as Mark Up that formats some of what you see, and References to other documents. Mark Up in HTML is made up of Tags. Tags are like instructions to tell a browser what Elements to use. Elements that Tags produce can be Inline or Block. Block Elements create an invisible box around the content while Inline Elements only work along single lines.

          INLINE TAGS: b (bold), br (break), em (italic),
a (anchor), img (image), span, strong

BLOCK TAGS: p (paragraph), div, form

LESSON ONE - HTML Structure

HTML STRUCTURE

      The standard structure of an HTML document is much like a tree. The tree branches off in multiple directions from a central sort of trunk. This structure is sometimes called a DOM or a Document Object Model. HTMLs structure is basically boxes inside of boxes inside of boxes, each box being an Element (Tag). Coders usually use indentation within the structure of the code they are writing. This isn't understood by the computer, but allows the code to have more visual organization for the coder.

CASCADING STYLE SHEETS (CSS)

      CSS controls the style of different HTML Elements. For example, this text is within a paragraph tag, and CSS is applying the font type and the text alignment for paragraph tag.

Back to Top

REPETITION

      Most objectives can be reached with slightly different code. One might apply apply a single CSS font to 10 of their elements and others to 2 elements, when it would have been more efficient to apply that font to a broader element (like the body) and then just apply the 2 other fonts to their proper elements. The first way involves needless repetition which wastes time and increases the likelihood of errors. The problem compounds further when things need to be changed or errors are made, as fixing them will also be more time consuming.
